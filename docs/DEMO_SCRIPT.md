# Demo Script

## 讲解结构

1. 介绍作品目标：帮助开发者在提交 PR 前完成结构化自检，减少空 PR 描述、测试缺失和高风险变更遗漏。
2. 展示主界面：左侧输入 PR 信息，中间是 Review 报告，右侧是变更文件统计。
3. 使用内置示例：说明该 PR 涉及仓库导入，diff 中有前端 token、删除测试、loading 状态等风险。
4. 展示审查意见：逐条讲解安全、测试、可靠性、可维护性和流程问题。
5. 展示 PR 描述生成：说明生成内容符合主办方要求的功能描述、实现思路和测试方式。
6. 展示测试与交付检查：说明 README 依赖、原创范围、demo 视频、主分支可运行等要求。
7. 介绍工程质量：核心逻辑有单元测试，项目支持 `npm run test`、`npm run build`、`npm run lint`。
8. 说明后续规划：接入 GitHub/Gitee PR URL、真实 LLM、团队规则和 Markdown 导出。

## 录屏提示

- 录制时打开浏览器控制台不是必须，重点展示可操作流程。
- 视频中需要有声音讲解。
- 上传到 bilibili、云盘等外部可访问平台后，把链接写回 README 的 Demo 区域。
